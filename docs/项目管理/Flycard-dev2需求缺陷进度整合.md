### 1.1 业务说明与更新边界

基于当前 `flycard-dev2` Git 提交可确认，FLYCARD/飞鸟探亲业务围绕“小程序端 + 后台管理端”两端展开：

- **小程序端**：承载用户侧信件服务、信件商品化、普通商品下单、购物车、订单详情、售后/退款展示、回信/客服、律师服务、律师分配展示、服务评价、微信订阅消息等用户链路。
- **后台管理端**：承载 FastAdmin 后台运营能力，包括信件配置、商品/订单管理、审核人员与订单审核、律师管理、律师分类、服务分配/进度、服务评论、支付退款、第三方登录、插件与初始化配置等运营链路。
- **模块口径**：当前 Bug 反馈系统中 FLYCARD 只保留 `小程序端` 与 `后台管理端` 两个启用模块；历史细分模块只作为 Git 整理来源，不再作为系统展示模块。

> 重要保护规则：后续如果继续更新 FLYCARD，只能补充或调整 Git 整理出来的需求、缺陷、迭代、里程碑、版本、模块归并、证据 Commit 与验收建议；不得因为导入脚本或整理文档而覆盖项目名称、负责人、项目标识、业务描述、项目阶段、计划时间、进度、风险等项目基础字段。
