## 1. 任务背景

当前 Bug 反馈系统已经具备项目、模块、版本、成员、Bug 工单、附件、评论、状态流转、统计看板等能力。现在需要继续实现项目管理功能，覆盖：

1. 项目阶段和项目进度管理。
2. 需求管理、需求评审、排期、验收。
3. 迭代计划和里程碑。
4. 老板/管理层驾驶舱。
5. 项目负责人和老板可查看当前处理、历史完成、未处理的需求和 Bug 数据。

## 3. 已确认产品范围

### 3.1 第一版必须实现

1. 项目阶段字段和阶段流转。
2. 项目计划开始/完成时间、风险等级、风险说明、进度字段。
3. 需求管理：创建、编辑、列表、详情、状态流转。
4. 需求关联项目、模块、版本、迭代、里程碑和 Bug。
5. 迭代管理：创建、编辑、状态、范围统计。
6. 里程碑管理：创建、编辑、目标日期、完成条件、延期识别。
7. 老板驾驶舱：全局项目进度、风险、延期、下阶段安排。
8. 当前处理事项：开发正在处理的需求、修复中的 Bug 数量与明细。
9. 历史完成事项：已完成需求、已修复 Bug 数量与明细。
10. 未处理事项：未处理需求、未处理 Bug 数量与明细。
11. 项目负责人概览：单项目细节和推进视图。
12. 权限和菜单种子数据。

### 3.2 第一版暂缓

1. 完整甘特图。
2. 拖拽看板状态流转。
3. WebSocket 实时推送。
4. 自动排期。
5. 工时日报和成本管理。
6. 管理层周报导出。
7. AI 项目风险分析。

## 5. 开发约束

1. 自有代码文件写入前必须 `wc -l` 检查行数。
2. 自有代码文件超过或接近 500 行必须拆分。
3. 不主动执行 `build`、`install`、打包、部署。
4. 不主动执行 Git 命令，除非用户当次明确要求。
5. 不主动操作 Docker 内测版或 `localhost:18001`。
6. 新接口必须加 Swagger、JWT、PermissionGuard、权限标识。
7. 后端 Service 禁止直接 `throw new Error()`，统一使用 `BusinessException`。
8. 状态流转和统计分组必须集中配置，避免页面硬编码。
9. 前端新页面使用 `<script setup lang="ts">`，逻辑拆分到组件、常量、API 和类型文件。
