# Bug 提交页附件面板线框图

最后更新时间：2026-05-14 18:20  
当前状态：线框草案，未改代码  
适用页面：`/bug/create`

## 设计目标

- 提交 Bug 时，开发人员最需要看清楚的是附件，尤其是标注图。
- 表单信息保持完整，但视觉优先级降低。
- 附件面板需要足够大，让标注图、文件名、类型、大小、说明、操作入口在同一区域内清楚呈现。
- 默认只上传标注图；需要时才保留原图。

## 推荐方案：左右分栏，附件为主

## 更新方案：主预览区 + 缩略图列表

根据新的交互要求，右侧附件面板建议从“单个大卡片”升级为 **附件浏览器**：

- 右侧面板左边是主预览区，显示当前选中的附件。
- 右侧面板右边是缩略图列表，显示所有已上传附件。
- 点击缩略图后，主预览区切换到对应附件。
- 主预览区支持左右切换上一张/下一张。
- 点击主预览区可打开全屏预览。
- 图片/标注图直接预览；视频在主预览区播放；日志/JSON/文本可显示文件摘要和“打开/下载”。

### 交互状态建议

| 状态 | 主预览区表现 | 缩略图列表表现 |
|---|---|---|
| 无附件 | 显示上传引导和示例说明 | 不显示或显示空状态 |
| 上传中 | 当前预览区显示进度/骨架屏 | 对应缩略图显示上传中 |
| 图片/标注图 | 大图完整展示，object-contain | 显示图片缩略图和类型标签 |
| 视频 | 主预览区显示 video 控件 | 显示视频封面/图标 |
| 日志/文本 | 显示文件图标、文件名、大小、打开按钮 | 显示文件类型图标 |
| 上传失败 | 显示失败原因和重新上传按钮 | 对应缩略图标红 |

### 我建议实现时的组件拆分

```text
AttachmentUploader
├─ AttachmentPreviewPanel        主预览区、左右切换、全屏入口
├─ AttachmentThumbnailList       缩略图列表、点击选中
└─ AttachmentFullscreenPreview   全屏预览弹层
```

这样后续 Bug 详情页也可以复用同一套附件浏览器，而不是提交页和详情页各写一套。

## 我建议采用的交互顺序

1. 用户先填写标题、项目、模块。
2. 立即上传/标注附件。
3. 附件面板显示大图和关键信息。
4. 问题描述、复现步骤、期望结果、实际结果保持非必填，可作为补充信息。
5. 提交前如果没有附件，可以允许提交，但提示“建议上传截图或日志帮助定位”。
