# 开发规范索引

本目录用于存放项目级开发规范和团队编码约定。这里记录“所有同类功能以后都应如何实现”的通用准则；单次故障复盘、问题根因和防重复排查记录放在 `../问题解决/`。

## 文档列表

| 文档 | 适用范围 |
|---|---|
| [前端页面交互与视觉规范.md](./前端页面交互与视觉规范.md) | 表格/列表刷新按钮、状态/风险语义色、数字指标颜色、老板驾驶舱和统计看板通用视觉准则 |
| [错误处理规范.md](./错误处理规范.md) | 前后端错误码、业务异常、错误响应格式 |
| [日志使用规范.md](./日志使用规范.md) | Winston 日志级别、日志格式、日志轮转和使用方式 |
| [时间处理规范.md](./时间处理规范.md) | UTC 存储、本地显示、前后端时间处理 |
| [测试策略.md](./测试策略.md) | 类型检查、冒烟测试、手动验收和发布前验证 |
| [ESLint配置指南.md](./ESLint配置指南.md) | ESLint、Prettier 配置和格式化约定 |

## 分类边界

- 放入本目录：全局开发规范、组件设计规范、视觉语义规范、编码约定、测试约定。
- 放入 `../问题解决/`：具体问题现象、根因、错误决策链路、防重复检查和排障经验。
- 如果某条问题总结沉淀为长期通用准则，应迁移到本目录，并从问题解决索引中移除重复正文。
