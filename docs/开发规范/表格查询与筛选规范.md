# 表格查询与筛选规范

**最后更新：** 2026-05-20  
**适用范围：** 后台管理页面、业务列表页、表格页、看板列表、统计明细列表中的搜索栏、筛选栏、表头筛选和查询参数设计。

## 1. 核心原则

表格搜索能力必须采用“**默认展示常用搜索项 + 展开后展示完整可查询项**”的分层设计。

- 默认搜索区只展示少量常用字段，避免首屏拥挤。
- 展开搜索区必须让用户看到当前表格所有支持查询的字段。
- 哪些字段默认展示，不能由开发人员主观猜测，必须有产品、业务、用户反馈或使用数据依据。
- 每个表头字段新增或调整时，都必须同步评估是否需要查询能力。
- 搜索控件必须按字段类型限制输入方式，不能所有字段都使用普通输入框。
- 数字、金额、耗时、日期、时间等区间型字段必须支持范围查询，并支持单边区间。
- 表格列配置、搜索配置、后端查询参数应尽量使用统一 schema 管理，避免表头与搜索栏脱节。

## 2. 搜索区结构规范

### 2.1 默认搜索区

默认搜索区用于展示用户最容易用到的查询项，通常包含：

- 关键词 / 编号 / 标题等基础检索字段
- 状态、类型、优先级等核心管理字段
- 创建时间、更新时间等关键时间字段
- 当前页面业务最核心的归属字段，例如项目、负责人、处理人

默认展示字段的确定依据优先级如下：

1. 产品需求明确指定。
2. 业务人员或真实用户明确反馈高频使用。
3. 已有使用数据能证明高频。
4. 当前页面核心业务流程中必须频繁使用。
5. 无明确依据时，只保留最基础的搜索项，其余放入展开搜索区。

> 开发人员不得因为“看起来常用”就主观决定默认展示字段。

### 2.2 展开搜索区

搜索栏必须提供清晰的“展开 / 收起”能力，例如：

- `展开筛选`
- `更多筛选`
- `高级筛选`
- `收起筛选`

展开后必须满足：

- 展示当前表格所有支持查询的字段。
- 默认区已有字段可以继续保留，也可以在展开态统一重排为完整筛选表单，但不能让用户找不到完整查询能力。
- 可查询但不常用的字段，应放入展开搜索区，而不是被省略。
- 不可查询字段必须在字段配置或需求说明中记录原因。

## 3. 表头字段与搜索字段关系

新增或修改表格列时，必须同步评估以下内容：

| 评估项 | 要求 |
|---|---|
| 是否展示在表格中 | 明确 `showInTable` 或等效配置 |
| 是否支持查询 | 明确 `search.enabled` |
| 是否默认展示 | 明确 `search.defaultVisible` |
| 查询控件类型 | 根据字段类型选择 input/select/range/dateRange 等 |
| 后端查询能力 | 明确对应 DTO、接口参数和数据库查询条件 |
| 不可查询原因 | 操作列、派生字段、性能成本、后端暂不支持等必须说明 |

常见不可查询字段示例：

- 操作列、快捷操作列。
- 纯前端展示字段。
- 临时计算字段。
- 后端暂未建索引且查询成本过高的字段。
- 当前业务没有查询价值的装饰性字段。

## 4. 字段类型与搜索控件规范

### 4.1 文本类型

适用字段：

- 标题
- 名称
- 编号
- 备注
- 描述

控件要求：

- 使用文本输入框。
- 输入内容应去除首尾空格。
- 匹配方式应按业务明确为包含、精确匹配或前缀匹配。
- 不允许把枚举、数字、时间字段简单做成文本输入框。

### 4.2 枚举类型

适用字段：

- 状态
- 类型
- 优先级
- 分类
- 风险等级

控件要求：

- 使用下拉单选或下拉多选。
- 选项必须来自统一枚举配置或后端字典接口。
- 禁止让用户手动输入枚举值。
- 前端展示值和后端提交值必须分离，避免把中文文案直接当作业务值提交。

### 4.3 数字类型

适用字段：

- 金额
- 数量
- 次数
- 分数
- 排序值
- 耗时
- 进度百分比

数字类型必须使用范围查询：

```text
最小值 ～ 最大值
```

查询规则：

| 用户输入 | 查询含义 |
|---|---|
| 只填最小值 | `字段 >= 最小值` |
| 只填最大值 | `字段 <= 最大值` |
| 同时填写最小值和最大值 | `最小值 <= 字段 <= 最大值` |
| 都不填写 | 不参与查询 |

校验要求：

- 输入框必须限制为数字。
- 最小值不能大于最大值。
- 小数位、负数、单位、百分比上限等规则必须按字段配置限制。
- 空值不应传给后端。

### 4.4 日期 / 时间类型

适用字段：

- 创建时间
- 更新时间
- 提交时间
- 处理时间
- 截止时间
- 完成时间

日期 / 时间类型必须使用范围查询：

```text
开始时间 ～ 结束时间
```

查询规则：

| 用户输入 | 查询含义 |
|---|---|
| 只填开始时间 | `字段 >= 开始时间` |
| 只填结束时间 | `字段 <= 结束时间` |
| 同时填写开始时间和结束时间 | `开始时间 <= 字段 <= 结束时间` |
| 都不填写 | 不参与查询 |

校验要求：

- 开始时间不能晚于结束时间。
- 日期型搜索如果用户只选择日期，结束日期必须包含当天完整范围，避免漏查当天数据。
- 推荐后端使用半开区间处理日期结束值：`>= 当天 00:00:00` 且 `< 次日 00:00:00`。
- 前后端必须统一时区处理规则，避免 UTC 存储和本地展示导致查询偏移。

示例：

```text
用户选择：2026-05-20 ～ 2026-05-20
实际含义：2026-05-20 00:00:00 ～ 2026-05-20 23:59:59
推荐后端条件：>= 2026-05-20 00:00:00 且 < 2026-05-21 00:00:00
```

### 4.5 布尔类型

适用字段：

- 是否启用
- 是否处理
- 是否公开
- 是否删除
- 是否归档

控件要求：

- 使用三态选择：全部 / 是 / 否。
- 默认值应为“全部”，表示不参与查询。
- 禁止使用文本输入框输入 `true`、`false`、`1`、`0`。

### 4.6 关联对象类型

适用字段：

- 项目
- 负责人
- 创建人
- 处理人
- 所属团队
- 关联需求

控件要求：

- 使用搜索选择器、远程下拉或弹窗选择器。
- 前端显示名称，后端提交稳定 ID。
- 如果数据量大，必须支持远程搜索和分页加载，不能一次性拉取全部数据。
- 标记重复、关联需求、选择用户等场景，不应让用户手动填写 ID。

## 5. 表头列筛选规范

部分字段可以额外支持表头列筛选，但不能替代搜索栏展开能力。

适合表头列筛选的字段：

- 状态
- 类型
- 分类
- 优先级
- 风险等级
- 是否启用

要求：

- 表头列筛选适合快速局部过滤。
- 展开搜索区仍应保留完整查询入口。
- 同一字段如果同时出现在搜索栏和表头筛选中，状态必须同步，不能出现两个筛选条件互相打架。

## 6. 前后端查询参数规范

### 6.1 空值处理

- 空字符串、空数组、`null`、`undefined` 不应作为有效查询条件提交。
- 重置筛选时必须清空默认区和展开区全部查询条件。
- 点击搜索后，分页应回到第一页。
- 点击刷新当前数据时，应保留当前搜索条件、分页条件、排序条件和展开状态。

### 6.2 范围参数

数字和时间范围建议使用统一命名方式，避免每个页面自定义一套参数。

示例：

```ts
{
  amountMin: 100,
  amountMax: 500,
  createdAtStart: '2026-05-01T00:00:00+08:00',
  createdAtEnd: '2026-05-20T23:59:59+08:00'
}
```

或使用后端统一约定的 `gte/lte` 结构：

```ts
{
  amount: {
    gte: 100,
    lte: 500
  },
  createdAt: {
    gte: '2026-05-01T00:00:00+08:00',
    lt: '2026-05-21T00:00:00+08:00'
  }
}
```

项目内应选择一种方式长期统一，不允许同类页面各自定义不兼容格式。

## 7. 推荐 schema 配置

表格列和搜索项建议统一配置，避免表头有字段但搜索栏遗漏。

```ts
const tableFields = [
  {
    key: 'title',
    label: '标题',
    showInTable: true,
    search: {
      enabled: true,
      defaultVisible: true,
      type: 'text',
      matchMode: 'contains'
    }
  },
  {
    key: 'status',
    label: '状态',
    showInTable: true,
    search: {
      enabled: true,
      defaultVisible: true,
      type: 'select',
      multiple: true,
      optionsSource: 'bugStatus'
    }
  },
  {
    key: 'amount',
    label: '金额',
    showInTable: true,
    search: {
      enabled: true,
      defaultVisible: false,
      type: 'numberRange',
      minKey: 'amountMin',
      maxKey: 'amountMax'
    }
  },
  {
    key: 'createdAt',
    label: '创建时间',
    showInTable: true,
    search: {
      enabled: true,
      defaultVisible: false,
      type: 'dateRange',
      startKey: 'createdAtStart',
      endKey: 'createdAtEnd',
      endInclusive: true
    }
  },
  {
    key: 'operation',
    label: '操作',
    showInTable: true,
    search: {
      enabled: false,
      reason: '操作列不对应业务查询字段'
    }
  }
]
```

## 8. 新增或改造表格前检查清单

开发新增或改造表格页面前，必须检查：

- [ ] 是否有默认搜索区。
- [ ] 是否有展开 / 收起筛选能力。
- [ ] 展开后是否能看到所有支持查询的字段。
- [ ] 默认展示字段是否有产品、业务、用户反馈或数据依据。
- [ ] 每个表头字段是否已评估查询能力。
- [ ] 不可查询字段是否写明原因。
- [ ] 数字字段是否使用范围查询。
- [ ] 时间字段是否使用范围查询，并正确处理单边区间和当天结束时间。
- [ ] 枚举字段是否使用下拉选择而不是文本输入。
- [ ] 布尔字段是否使用全部 / 是 / 否三态选择。
- [ ] 关联对象字段是否使用选择器并提交稳定 ID。
- [ ] 搜索、重置、刷新、分页、排序之间的状态是否一致。
- [ ] 前后端查询参数命名是否符合项目统一约定。

## 9. 一句话规范

表格搜索栏必须采用“默认常用搜索项 + 展开全部可查询项”的结构；每个表头字段都要评估查询能力；搜索控件必须根据字段类型进行限制，数字和时间字段必须支持范围查询，并允许只填写起始值或结束值形成单边区间查询。
