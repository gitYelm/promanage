# 新项目初始化指南

基于 RBAC Admin Pro 脚手架创建新项目时，请按以下清单进行配置替换。

---

## 🚀 快速开始

### 1. 克隆并重命名项目

```bash
# 克隆脚手架
git clone <repo-url> your-project-name
cd your-project-name

# 删除原有 git 历史
rm -rf .git
git init
```

### 2. 创建本地数据库

需要 PostgreSQL 超级管理员账号（通常是 `postgres`）：

```bash
# 连接 PostgreSQL（需要超管用户名和密码）
psql -U postgres -h localhost
# 或指定密码
PGPASSWORD=your_superuser_password psql -U postgres -h localhost

# 创建数据库和用户
CREATE DATABASE your_project_db;
CREATE USER your_db_user WITH ENCRYPTED PASSWORD 'YourPassword@2024';
GRANT ALL PRIVILEGES ON DATABASE your_project_db TO your_db_user;

# 授予 schema 权限（PostgreSQL 15+ 需要）
\c your_project_db
GRANT ALL ON SCHEMA public TO your_db_user;

# 退出
\q
```

或使用 Docker 快速启动（无需本地 PostgreSQL）：

```bash
# 修改根目录 .env 后启动
docker-compose up -d postgres redis
```

## 📝 必须修改的配置

### 1. 项目基础信息

| 文件 | 配置项 | 说明 |
|------|--------|------|
| `package.json` | `name` | 项目名称 |
| `web/package.json` | `name` | 前端包名 |
| `server-nestjs/package.json` | `name` | 后端包名 |

### 2. 环境变量配置

#### 根目录 `.env`
```properties
POSTGRES_DB=your_project_db        # 数据库名
POSTGRES_USER=your_db_user         # 数据库用户
POSTGRES_PASSWORD=YourPassword     # 数据库密码（生产环境使用强密码）
JWT_SECRET=<生成新密钥>             # 执行: node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

#### 后端 `server-nestjs/.env`
```properties
DATABASE_URL="postgresql://用户名:密码@localhost:5432/数据库名?schema=public"
JWT_SECRET=<与根目录保持一致>
```

### 3. 网站信息（数据库配置）

通过 **系统管理 > 参数设置** 或直接修改种子数据：

| 配置键 | 说明 | 示例 |
|--------|------|------|
| `sys.app.name` | 网站名称 | `我的管理系统` |
| `sys.app.description` | 网站描述 | `企业级后台管理系统` |
| `sys.app.copyright` | 版权信息 | `© 2025 Your Company` |
| `sys.app.icp` | ICP备案号 | `京ICP备XXXXXXXX号` |
| `sys.app.logo` | 网站Logo | `/uploads/logo.png` |
| `sys.app.email` | 联系邮箱 | `admin@yourcompany.com` |

## 🔐 初始数据替换

### 1. 部门结构 (`server-nestjs/prisma/seed.ts`)

替换示例组织架构为你的实际组织：

```typescript
// 修改前
const rootDept = await ensureDept({
  deptName: '总公司',
  leader: '张总',
})

// 修改后
const rootDept = await ensureDept({
  deptName: '你的公司名称',
  leader: '实际负责人',
})
```

### 2. 初始用户

修改 `seed.ts` 中的用户信息：

```typescript
const adminUser = await ensureUser({
  userName: 'admin',
  nickName: '超级管理员',
  email: 'admin@yourcompany.com',      // 修改邮箱
  phonenumber: '13800000000',          // 修改手机号
})
```

### 3. 默认密码

修改初始密码（`seed.ts` 和 `db/init_data.sql`）：

```typescript
// seed.ts
const hashedPassword = await bcrypt.hash('你的初始密码', salt)
```

同时更新数据库配置：
```sql
-- sys_config 表
UPDATE sys_config SET config_value = '你的初始密码' WHERE config_key = 'sys.account.initPassword';
```

| 第 30 行 | `'© 2025 RBAC Admin Pro. All rights reserved.'` | 版权信息默认值 |
| 第 56 行 | `features` 数组 | 左侧特性介绍（4个卡片） |
| 第 178-179 行 | `'开箱即用的后台管理解决方案'` | 左侧大标题 |
| 第 181 行 | `'基于 Vue 3 + NestJS + PostgreSQL...'` | 左侧副标题 |
| 第 73、77 行 | `'admin'` / `'admin123'` | 开发环境默认填充（验证码关闭时显示） |

```typescript
// 1. 修改默认值（第 27-30 行）

  { icon: BarChart3, title: '你的特性4', desc: '特性4描述' },

```

```vue
<!-- 3. 修改左侧大标题（第 178-181 行） -->
<h2 class="text-3xl font-bold mb-4">你的系统<br />宣传语</h2>
```

## 🗄️ 数据库初始化

### 方式一：使用 Prisma（推荐）

```bash
cd server-nestjs

# 1. 生成 Prisma Client
pnpm db:generate

# 2. 执行迁移
pnpm db:migrate

# 3. 初始化种子数据
pnpm db:seed
```

### 方式二：使用 SQL 脚本

```bash
# 1. 创建表结构
psql -U your_user -d your_db -f db/schema.sql

# 2. 初始化数据
psql -U your_user -d your_db -f db/init_data.sql
```

## 🔧 可选配置

### 邮件服务

```properties
# sys_config 表
sys.mail.enabled=true
sys.mail.host=smtp.yourmail.com
sys.mail.port=465
sys.mail.username=noreply@yourcompany.com
sys.mail.password=your_password
sys.mail.from=noreply@yourcompany.com
```

### 文件存储

```properties
# 本地存储
sys.storage.type=local
sys.storage.local.path=./uploads

# OSS 存储
sys.storage.type=oss
sys.storage.oss.endpoint=oss-cn-hangzhou.aliyuncs.com
sys.storage.oss.bucket=your-bucket
sys.storage.oss.accessKey=your_access_key
sys.storage.oss.secretKey=your_secret_key
```

### 安全设置

```properties
sys.account.captchaEnabled=true      # 启用验证码
sys.account.twoFactorEnabled=true    # 启用两步验证
sys.login.maxRetry=5                 # 登录失败次数限制
sys.login.lockTime=10                # 账户锁定时长（分钟）
```

## 📌 注意事项

1. **生产环境**：务必修改所有默认密码和密钥
2. **CORS 配置**：生产环境需配置 `CORS_ORIGINS`
3. **Redis**：多实例部署需启用真实 Redis
4. **备份**：修改前备份原始配置文件
