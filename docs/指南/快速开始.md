## 本地开发

适用于开发调试场景。本地开发只用 Docker 跑数据库，前后端代码直接本地运行（支持热重载）。

### 1. 启动数据库（Docker）

```bash
# 只启动 PostgreSQL + Redis，不启动应用服务
docker compose up -d postgres redis

# 或使用项目脚本
./db.sh start
```

> 💡 不要用 `docker compose up -d` 启动全部服务，那样会把前后端也容器化，改代码不方便。

### 2. 初始化数据库

```bash
# 使用 Prisma（推荐）
cd server-nestjs
pnpm prisma migrate deploy
pnpm prisma db seed
```

### 3. 配置环境变量

```bash
# 后端配置
cd server-nestjs
cp .env.example .env
# 编辑 .env，确保 DATABASE_URL 正确
```

### 4. 启动服务

```bash
# 在项目根目录，同时启动前后端
pnpm dev

# 或分别启动
pnpm dev:web     # 前端 http://localhost:5173
pnpm dev:server  # 后端 http://localhost:3000
```

### 5. 访问系统

| 服务 | 地址 |
|------|------|
| 前端 | http://localhost:5173 |
| 后端 API | http://localhost:3000 |
| API 文档 | http://localhost:3000/api-docs |

默认账号：`admin` / `admin123`

## 手动部署（生产环境）

适用于不使用 Docker、需要更灵活配置的场景。

### 1. 数据库准备

#### 安装 PostgreSQL

```bash
# Ubuntu/Debian
sudo apt install postgresql-16
sudo systemctl start postgresql

# CentOS/RHEL
sudo dnf install postgresql16-server
sudo postgresql-setup --initdb
sudo systemctl start postgresql
```

#### 创建数据库

```bash
sudo -u postgres psql

CREATE USER rbac_admin WITH PASSWORD 'YourStrongPassword';
CREATE DATABASE rbac_admin OWNER rbac_admin;
GRANT ALL PRIVILEGES ON DATABASE rbac_admin TO rbac_admin;
\q
```

#### 初始化数据

```bash
cd server-nestjs
pnpm prisma migrate deploy
pnpm prisma db seed
```

### 2. Redis 准备（生产环境建议启用）

```bash
# Ubuntu/Debian
sudo apt install redis-server
sudo systemctl start redis

# 验证
redis-cli ping
```

### 3. 后端部署

```bash
cd server-nestjs

# 安装依赖
pnpm install

# 配置环境变量
cp .env.example .env
vim .env
```

关键配置：

```bash
NODE_ENV=production
PORT=3000

# 数据库（密码中的特殊字符需 URL 编码）
DATABASE_URL="postgresql://rbac_admin:YourPassword@localhost:5432/rbac_admin"

# JWT 密钥（必须修改！）
JWT_SECRET=your-32-char-random-string

# Redis
REDIS_ENABLED=true
REDIS_URL=redis://127.0.0.1:6379

# CORS
CORS_ORIGINS=https://your-domain.com
```

```bash
# 生成 Prisma Client
pnpm prisma generate

# 构建
pnpm build

# 启动（推荐使用 PM2）
pm2 start dist/main.js --name rbac-server
```

### 4. 前端部署

```bash
cd web

# 安装依赖
pnpm install

# 构建
pnpm build
```

将 `dist` 目录部署到 Web 服务器。

### 5. Nginx 配置

```nginx
server {
    listen 80;
    server_name your-domain.com;
    
    # Gzip
    gzip on;
    gzip_types text/plain text/css application/javascript application/json;

    root /var/www/rbac-admin/dist;
    index index.html;

    # SPA 路由
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 代理
    location /api/ {
        proxy_pass http://127.0.0.1:3000/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff2?)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

## 生产环境安全配置

### 安全清单

| 配置项 | 说明 |
|--------|------|
| `POSTGRES_PASSWORD` | 使用强密码（16+ 字符，含特殊字符） |
| `JWT_SECRET` | 至少 32 位随机字符串 |
| `CORS_ORIGINS` | 配置实际域名，不要用 `*` |
| `NODE_ENV` | 设为 `production` |
| `ERROR_FULL` | 设为 `false`，隐藏详细错误 |

### 生成安全密钥

```bash
# JWT 密钥
openssl rand -hex 32

# 或使用 Node.js
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

### HTTPS 配置

```nginx
server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate /etc/nginx/ssl/your-domain.crt;
    ssl_certificate_key /etc/nginx/ssl/your-domain.key;
    ssl_protocols TLSv1.2 TLSv1.3;

    # ... 其他配置同上
}

server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}
```

### 数据备份

```bash
# 备份
pg_dump -U rbac_admin rbac_admin > backup_$(date +%Y%m%d).sql

# 恢复
psql -U rbac_admin rbac_admin < backup_20241210.sql
```

## 常见问题

### 数据库连接失败

检查 `DATABASE_URL` 格式，密码中的特殊字符需 URL 编码：
- `@` → `%40`
- `#` → `%23`
- `:` → `%3A`

### 前端无法访问 API

1. 检查 `CORS_ORIGINS` 配置
2. 检查 Nginx 代理配置
3. 确认后端服务正常：`curl http://localhost:3000/health`

### Redis 连接失败

开发环境可设置 `REDIS_ENABLED=false` 使用内存模式。

### PM2 常用命令

```bash
pm2 start dist/main.js --name rbac-server  # 启动
pm2 restart rbac-server                     # 重启
pm2 logs rbac-server                        # 查看日志
pm2 monit                                   # 监控
```
