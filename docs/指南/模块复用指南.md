# 模块复用指南

基于同一脚手架的项目之间快速复用业务模块。

## 使用方式

1. 在 IDE 中打开两个项目的工作区（源项目 + 目标项目）
2. 复制下方提示词，替换模块名称
3. 让 AI 执行复制操作

---

## 提示词模板

### 完整模块复制（后端 + 前端 + 数据库）

```
请将源项目的 [模块名] 模块完整复制到目标项目。

源项目路径: [/path/to/source-project]
目标项目路径: [/path/to/target-project]
模块名称: [app-config]

请按以下步骤执行:

1. **后端代码复制**
   - 复制 `server-nestjs/src/admin/[模块名]/` 整个目录
   - 检查模块中的 import 语句，确保引用的公共代码在目标项目中存在
   - 如果引用了 `@/common/` 下的装饰器、守卫、拦截器等，确认目标项目有相同实现
   - **重要**: 如果目标项目缺少公共代码，需要一并复制（参考下方公共代码依赖清单）

2. **前端代码复制**
   - 复制 `web/src/views/[模块名]/` 目录（页面）
   - 复制 `web/src/api/[模块名]/` 目录（API 接口）
   - 检查是否使用了 `@/components/common/` 或 `@/components/business/` 下的组件
   - **重要**: 如果目标项目缺少公共组件，需要一并复制

3. **Prisma Model**
   - 从源项目 `server-nestjs/prisma/schema.prisma` 中提取相关 Model
   - 追加到目标项目的 schema.prisma
   - 列出需要执行的命令: pnpm db:migrate 和 pnpm db:generate

4. **种子数据 (Seed)**
   - 从源项目 `server-nestjs/prisma/seed.ts` 中提取相关的种子数据逻辑
   - 复制到目标项目的 seed.ts 中
   - 包含菜单、字典等初始化数据

5. **模块注册**
   - 后端: 在目标项目的 `server-nestjs/src/app.module.ts` 中导入新模块
   - 前端: 路由是动态生成的，只需复制 views 和 api 文件，菜单通过 seed 数据配置

6. **输出清单**
   - 列出所有复制的文件（包括公共代码）
   - 列出需要运行的命令
```

### 仅后端模块

```
请将源项目的 [模块名] 后端模块复制到目标项目。

源项目: [/path/to/source-project]
目标项目: [/path/to/target-project]

复制内容:
1. server-nestjs/src/admin/[模块名]/ 目录
2. 相关的 Prisma Model
3. 相关的种子数据 (seed.ts 中的菜单权限)

注意事项:
- 检查 import 依赖，确保公共代码存在
- 输出需要手动执行的步骤
```

### 仅前端页面

```
请将源项目的 [模块名] 前端页面复制到目标项目。

源项目: [/path/to/source-project]
目标项目: [/path/to/target-project]

复制内容:
1. web/src/views/[模块名]/ 目录
2. web/src/api/[模块名]/ 目录

注意事项:
- 检查组件依赖是否存在
- 检查 API 类型定义
```

## 公共代码依赖清单

复制模块时需要确认目标项目存在以下公共代码，**如果缺失则需要一并复制**：

### 后端公共代码 (`server-nestjs/src/common/`)

| 路径 | 说明 | 常用于 |
|------|------|--------|
| `decorators/log.decorator.ts` | @Log 操作日志装饰器 | Controller |
| `decorators/permission.decorator.ts` | @RequirePermission 权限装饰器 | Controller |
| `decorators/data-scope.decorator.ts` | @DataScope 数据权限装饰器 | Controller |
| `guards/permission.guard.ts` | 权限守卫 | 全局 |
| `interceptors/transform.interceptor.ts` | 响应转换拦截器 | 全局 |
| `interceptors/operlog.interceptor.ts` | 操作日志拦截器 | 全局 |
| `exceptions/business.exception.ts` | 业务异常类 | Service |
| `enums/error-code.enum.ts` | 错误码枚举 | Service |
| `utils/` | 工具函数 | 各处 |

### 前端公共代码 (`web/src/`)

| 路径 | 说明 |
|------|------|
| `components/ui/` | shadcn-vue 基础组件 |
| `components/common/TablePagination.vue` | 分页组件 |
| `components/common/TableSkeleton.vue` | 表格骨架屏 |
| `components/common/EmptyState.vue` | 空状态 |
| `components/common/ImageUpload.vue` | 图片上传 |
| `utils/request.ts` | Axios 封装 |
| `utils/format.ts` | 格式化函数 |
| `utils/auth.ts` | 认证工具 |
| `directive/permission.ts` | v-permission 指令 |

## 复制后检查清单

- [ ] 后端代码编译通过 (`pnpm build`)
- [ ] 前端代码编译通过 (`pnpm build`)
- [ ] Prisma Client 已更新 (`pnpm db:generate`)
- [ ] 数据库迁移已执行 (`pnpm db:migrate`)
- [ ] 种子数据已更新并执行 (`pnpm db:seed`)
- [ ] 页面可正常访问
- [ ] CRUD 功能正常

## 示例：复制 app-config 模块

```
请将 rbac-admin-pro 项目的 app-config 模块复制到 my-new-project。

源项目: /Users/xxx/projects/rbac-admin-pro
目标项目: /Users/xxx/projects/my-new-project

这是一个 App 配置管理模块，包含:
- App 基础配置
- 版本管理
- 启动页管理
- 协议管理
- 广告管理
- 反馈管理

请完整复制后端、前端代码，并输出:
1. Prisma Model 定义
2. 种子数据 (seed.ts 中的菜单初始化代码)
3. 需要执行的命令清单
```
